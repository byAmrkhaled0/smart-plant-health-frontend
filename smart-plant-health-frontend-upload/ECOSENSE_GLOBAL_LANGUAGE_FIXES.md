# Ecosense AI - Global Language System Fixes

## Scope
This update fixes language behavior inside the same current Ecosense AI / Smart Plant Health project only.

## What changed
- Language is now applied globally through the shared language context.
- The selected language is saved in `localStorage` using `sph_lang` and persists after refresh.
- `document.documentElement.lang`, `document.documentElement.dir`, and `document.body.dir` are updated whenever language changes.
- RTL class is applied to the body when Arabic is selected.
- Diagnosis form options now display translated crop and light-level labels while keeping backend values stable.
- Diagnosis result labels remain translated across final_status, diagnosis, recommendations, actions, confidence, image_status, and sensor_status sections.
- Dynamic backend/model text now supports bilingual display mapping:
  - English API text is shown in Arabic when Arabic is selected.
  - Arabic API text is shown in English when English is selected where a known mapping exists.
- Backend Arabic fields such as recommendations_ar/actions_ar are supported when present.
- final_status remains the primary status for diagnosis, reports, alerts, and cards.
- sensor_status and image_status remain explanatory only.
- Notification messages generated from diagnosis use translation keys instead of fixed Arabic text.
- Duplicate safety_layer object key was removed from normalized diagnosis result.

## Build
`npm run build` completed successfully.

## Notes
Some backend diagnosis content may still arrive as free text. Known disease names, recommendations, statuses, and model messages are mapped locally. For full bilingual backend content, the backend should ideally return English and Arabic fields for all diagnosis text.
